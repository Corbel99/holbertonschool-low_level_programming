# Benchmarking

## 📖 Description

This project is part of the **Holberton School Low-Level Programming** curriculum.

The objective is to introduce the fundamentals of **benchmarking** and software efficiency by measuring the execution time of C programs under controlled conditions.

Rather than optimizing code, the project focuses on collecting reliable measurements, comparing implementations, and understanding how algorithmic efficiency can impact execution time and, indirectly, energy consumption.

## 🎯 Learning Objectives

By completing this project, I learned to:

* Compile C programs using strict compiler flags.
* Measure execution time using the C Standard Library.
* Compare different implementations under identical conditions.
* Understand the relationship between runtime, algorithmic efficiency, and sustainability.
* Analyze and interpret benchmark results.
* Write concise technical reports based on collected data.

## 🛠️ Compilation

All programs are compiled with:

```bash
gcc -Wall -Werror -Wextra -pedantic -std=gnu89 -Wno-long-long
```

## 📂 Project Structure

Each task explores a different aspect of benchmarking and performance analysis.
